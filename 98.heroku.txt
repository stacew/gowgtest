> go get -u github.com/tools/godep

--- 생성 --- 작업 폴더로 이동해서 heroku create 또는 Procfile 생성 등...
> heroku login
- 헤로쿠 워크스페이스 만들기 : heroku 사이트 대시보드에서 만들면 이름 지정 가능.
자신 프로젝트 git에 remote 주소 추가 heroku, https://git.heroku.com/"워크스페이스이름.git"
- Profile 추가
web: "프로젝트 이름"

--- 업데이트 --- 작업 폴더로 이동
> godep save ./...
디펜던시 갱신이 필요하면 잡아주기
- master를 commit 해줘야함. vendor까지
> git push heroku master 
빌드 + 실행까지 함.

--- 사용 --- 작업 폴더로 이동
> heroku open
하면 브라우저 열림
> heroku logs --tail
로그를 뒤에서부터 보고 싶다


> heroku ps
bin/프로젝트이름(1) <- 프로세스 개수 알 수 있음.
> heroku ps:scale web=2
2개 비용 올림. free버전은 불가능
0개로 하면 서비스 멈춤.
